ShellLab

这是我们这学期计算机系统课（CSAPP）的一个实验，由CMU引进，这个实验是让我们实现一个简易的shell，不用实现管道等高级功能，只需要实现运行新程序、bg/fg(让前后台程序继续运行)、quit、jobs(列出所有作业)。

需要细心读完书第七第八章节。要对书上例子熟读牢记。如果对书上的内容掌握的很好，这个实验就会比较简单，可以说是基本由自己去做一个完整的 shell 的功能(除了处理分割形成argv数组和jobs作业组)。锻炼的其实是大局观。如果有书本函数作为参考应该说会容易一些。

实验流程

补全tsh.c中剩余的代码：

  • void eval(char *cmdline)：解析并执行命令。
  • int builtin_cmd(char **argv)：检测命令是否为内置命令quit、fg、bg、jobs。
  • void do_bgfg(char **argv)：实现bg、fg命令。
  • void waitfg(pid_t pid)：等待前台命令执行完成。
  • void sigchld_handler(int sig)：处理SIGCHLD信号，即子进程停止或终止。
  • void sigint_handler(int sig)：处理SIGINT信号，即来自键盘的中断ctrl-c。
  • void sigtstp_handler(int sig)：处理SIGTSTP信号，即终端停止信号ctrl-z。

辅助函数(Lab本身自带供我们使用的函数)

int parseline(const char cmdline,char *argv)：获取参数列表char **argv，返回是否为后台运行命令（true）。

  • void clearjob(struct job_t *job)：清除job结构。
  • void initjobs(struct job_t *jobs)：初始化jobs链表。
  • void maxjid(struct job_t *jobs)：返回jobs链表中最大的jid号。
  • int addjob(struct job_t *jobs,pid_t pid,int state,char *cmdline)：在jobs链表中添加job
  • int deletejob(struct job_t *jobs,pid_t pid)：在jobs链表中删除pid的job。
  • pid_t fgpid(struct job_t *jobs)：返回当前前台运行job的pid号。
  • struct job_t *getjobpid(struct job_t *jobs,pid_t pid)：返回pid号的job。
  • struct job_t *getjobjid(struct job_t *jobs,int jid)：返回jid号的job。
  • int pid2jid(pid_t pid)：将pid号转化为jid。
  • void listjobs(struct job_t *jobs)：打印jobs。
  • void sigquit_handler(int sig)：处理SIGQUIT信号。

实验要求

  • tsh的提示符为tsh>
  • 用户的输入分为第一个的name和后面的参数，之间以一个或多个空格隔开。如果name是一个tsh内置的命令，那么tsh应该马上处理这个命令然后等待下一个输入。否则，tsh应该假设name是一个路径上的可执行文件，并在一个子进程中运行这个文件（这也称为一个工作、job）
  • tsh不需要支持管道和重定向
  • 如果用户输入ctrl-c (ctrl-z)，那么SIGINT (SIGTSTP)信号应该被送给每一个在前台进程组中的进程，如果没有进程，那么这两个信号应该不起作用。
  • 如果一个命令以“&”结尾，那么tsh应该将它们放在后台运行，否则就放在前台运行（并等待它的结束）
  • 每一个工作（job）都有一个正整数PID或者job ID（JID）。JID通过”%”前缀标识符表示，例如，“%5”表示JID为5的工作，而“5”代笔PID为5的进程。
  • tsh应该回收（reap）所有僵尸孩子，如果一个工作是因为收到了一个它没有捕获的（没有按照信号处理函数）而终止的，那么tsh应该输出这个工作的PID和这个信号的相关描述。

注意事项

进程和进程之间的的时间线是不同的，可能会发生这个进程已经完成但主进程还没将此进程记录等的情况，所以必须要保证有一些函数的顺序是正确的，而且函数在访问全局变量的时候，全局变量不会因为另一个进程的某些操作导致出现更改(这里主要是信号会触发信号处理函数)，这点就需要用到信号阻塞的方法，设置合理的阻塞，使得函数顺序合理，并且所有能得到的全局变量都是正确的。

实验代码

需要增加一个全局变量

eval

本实验最重要的函数，也是所有函数都要在这里被调用

builtin_cmd

判断是否是内置命令函数，判断quit、bg、fg、jobs这四个内置命令

waitfg

等待前台进程结束函数，这里函数可以参考书上p546页(中文书)，有详细的教程，运用了一个不可间断的安全函数sigsuspend来确保能接收SIGCHLD，并且接收后又不可间断地屏蔽回去

sigchld_handler

SIGCHLD信号处理函数，需要完成当有SIGCHLD信号发送到shell进程时，用于处理其是哪个子进程发出的，并且判断此子进程是停止的还是终止的(正常终止或因信号终止)，这里沿用书本waitfg方法，需要判断如果是前台程序的话要更改一个全局变量fg_pid，使得waitfg循环结束

sigint_handler

SIGINT信号处理函数，当接收到SIGINT信号时将其发送给前台进程

sigtstp_handler

SIGTSTP信号处理函数，当接收到SIGTSTP信号时将其发送给前台进程

do_bgfg

发送SIGCONT给指定pid(直接pid号)或jobid(%加jobid号)的进程，使它能继续运行在前台或后台
